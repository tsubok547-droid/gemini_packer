# Gemini Packer  packer️



大規模言語モデル（LLM）へソースコードを渡す際の、面倒な準備作業を自動化するためのGUIアプリケーションです。プロジェクトのファイルを選択し、複数のZIPファイルとAIへの指示プロンプトをワンクリックで生成します。

---

## ## 主な機能

* 📁 **フォルダのドラッグ＆ドロップ**: プロジェクトフォルダをウィンドウにドラッグ＆ドロップするだけで、ファイルツリーを瞬時に読み込みます。
* ✅ **直感的なファイル選択**:
    * チェックボックスをクリックして、AIに渡したいファイルやフォルダを直感的に選択できます。
    * フォルダをチェックすれば、その中のファイル/フォルダがすべて選択されます。
    * フォルダ内の一部のファイルだけを選択すると、親フォルダは「部分的選択」として表示されます。
* 💾 **選択状態の保存と復元**:
    * 現在の選択状態を`.json`ファイルにキャッシュとして保存できます。
    * 「フォルダ全体を選択した」という意図と「特定のファイルだけを選択した」という意図を厳密に区別して保存・復元するため、プロジェクトの構成が変わっても意図した通りの選択が復元されます。
* 📜 **構成ファイルの生成**: 選択したファイル/フォルダの階層構造を`directory_structure.txt`として出力します。AIにプロジェクトの全体像を伝えるのに役立ちます。
* 🤖 **ZIPとプロンプトの自動生成**:
    * 選択したファイルを指定した数（デフォルト: 10個）ごとに分割し、複数のZIPアーカイブを生成します。
    * 分割したZIPファイルをAIに順番に渡すための、コピー＆ペーストしやすい`prompts.txt`を同時に生成します。

## ## 使い方

### ### 実行ファイル (`gemini_packer.exe`) を使う場合

1.  リリースページから最新の`gemini_packer.exe`をダウンロードします。
2.  `gemini_packer.exe`をダブルクリックして起動します。
3.  分析したいプロジェクトフォルダをウィンドウ上にドラッグ＆ドロップします。
4.  ファイルツリーでAIに渡したいファイルやフォルダを選択します。
5.  必要に応じて「💾 現在の選択を保存」や「📜 構成ファイル生成」ボタンを使用します。
6.  「✅ 選択したファイルをZIP化」ボタンを押すと、プロジェクトフォルダ内に`gemini_files`というフォルダが作成され、ZIPファイルと`prompts.txt`が出力されます。

### ### ソースコードから直接実行する場合

1.  **前提条件**:
    * Python 3 がインストールされていること。
2.  **セットアップ**:
    * このリポジトリをクローンまたはダウンロードします。
    * ターミナルで以下のコマンドを実行し、必要なライブラリをインストールします。
        ```bash
        pip install tkinterdnd2
        ```
3.  **実行**:
    * ターミナルで以下のコマンドを実行してアプリケーションを起動します。
        ```bash
        python gemini_packer.py
        ```

## ## ビルド方法 (自分用の.exeを作成する)

このアプリケーションを自分自身でビルド（.exe化）することも可能です。

1.  **PyInstallerのインストール**:
    ```bash
    pip install pyinstaller
    ```
2.  **ビルドコマンドの実行**:
    プロジェクトのルートディレクトリ（`gemini_packer.py`と`assets`フォルダがある場所）で、以下のコマンドを実行します。
    ```bash
    pyinstaller --onefile --windowed --add-data "assets;assets" gemini_packer.py
    ```
3.  **完成**:
    コマンドが完了すると`dist`フォルダが作成され、その中に`gemini_packer.exe`が格納されています。
